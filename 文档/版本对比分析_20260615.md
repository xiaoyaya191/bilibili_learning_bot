## 三、版本间核心差异对比

### 3.1 API 限流机制（全部相同）

| 特性 | 四个版本 |
|------|---------|
| `_bili_throttle` | ✅ 全部：全局冷却 + 间隔节流 + 抖动 + session 首免 |
| `_bili_trigger_cooldown` | ✅ 全部：90~180s 全局冷却 |
| `_BILI_API_MIN_GAP` | ✅ 全部：`config.speed.api_min_gap` 默认 0.3s |
| `_api_with_retry` 退避 | ✅ 全部：`2^(attempt+1) * [2, 3.5]s` 指数退避 |
| 启动冷却 | ✅ 全部：5~10s (`COOLDOWN_STARTUP_MIN/MAX`) |
| `report_history` 节流 | ✅ 全部：`await _bili_throttle("上报历史")` + 指数退避 |
| `get_hot_comments` | ✅ 全部：range(4) + 长退避 `15+attempt*20` |

**结论：四个版本的限流逻辑几乎完全一致，差异在于代码组织结构。**
